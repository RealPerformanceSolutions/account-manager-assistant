# Draft a forward

Turn one of those small inbound requests (an engineer reporting a fault, a customer chasing service)
into a tidy forward to the right place, ready for the user to send. Work in their workspace.

## Get the request
- Locate the email via the Outlook connector, or ask the user to paste it.

## Work out where it goes
- If the user has mapped a workflow for this (`context/workflows/service-request.md` or similar),
  follow it for the destination and any standard wording.
- Otherwise ask who it should go to (e.g. the service department) and remember it for next time.

## Draft
- Write a short covering line in the user's voice (see `context/voice-profile.md`): what's needed,
  who from, any urgency, and what the user is asking the recipient to do.
- Keep the original message underneath so the recipient has the full context.

## Save & record
- Save to `drafts/YYYY-MM-DD-forward-<topic>.md`.
- Note it against the relevant account if there is one.
- Show the draft and remind the user to review and forward it themselves. Never send.
